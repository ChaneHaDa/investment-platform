# 디자인 개편 PRD

- 문서 버전: v1.0
- 작성일: 2026-02-13
- 대상 서비스: `stock-portfolio-backtest-front`
- 작성 목적: 현재 UI/UX 품질 이슈를 해결하고, 일관된 디자인 시스템 기반으로 유지보수성과 접근성을 개선한다.

## 1. 배경 및 문제 정의

현재 프로젝트는 기능은 동작하지만, 화면별 스타일 규칙이 혼재되어 다음 문제가 발생한다.

- 디자인 토큰/컬러 체계가 일관되지 않다.
- 일부 화면에서 모바일 반응형이 깨진다(고정 폭 모달, 12컬럼 폼 테이블).
- 다크모드 관련 전역 스타일 충돌 가능성이 있다.
- 상호작용 요소 시맨틱/접근성 품질이 낮다.
- 결과 차트 일부가 기본적으로 가로 스크롤을 유발한다.

이로 인해 사용자 신뢰도, 사용성, 변경 비용이 동시에 악화되고 있다.

## 2. 목표

이번 개편의 목표는 다음 4가지다.

1. 시각 일관성: 토큰 기반 색상/타이포/간격 체계 단일화
2. 사용성: 모바일 포함 주요 사용자 플로우의 조작성 개선
3. 접근성: 키보드 포커스/시맨틱/명도 대비 개선
4. 유지보수성: 공통 컴포넌트와 스타일 규칙으로 중복 제거

## 3. 비목표(Non-Goals)

- 백엔드 API 스펙 변경
- 신규 투자 기능 추가
- 다국어(i18n) 도입
- 차트 라이브러리 교체

## 4. 대상 사용자 및 핵심 시나리오

### 대상 사용자

- 개인 투자자(모바일/데스크톱 혼용)
- 기존 로그인 사용자(포트폴리오 관리, 백테스트)
- 비로그인 사용자(탐색 및 체험)

### 핵심 시나리오

1. 홈 → 로그인/회원가입 → 포트폴리오 생성
2. 포트폴리오 구성 → 백테스트 실행 → 결과 확인/저장
3. 주식 검색 → 상세 확인 → 포트폴리오 반영

## 5. 범위

### 포함 범위(In Scope)

- `app/globals.css`, `tailwind.config.ts` 디자인 토큰 체계 정리
- 헤더/푸터/메인 레이아웃 일관화
- 폼/버튼/카드/테이블/모달 공통 규칙 정립
- 백테스트/포트폴리오/주식 페이지 반응형 개편
- 접근성 개선(포커스 상태, 시맨틱, 대비)
- 차트 레이아웃 가독성 개선

### 제외 범위(Out of Scope)

- API 연동 로직 전면 리팩터링
- 인증 구조 변경
- 대규모 정보구조(IA) 재설계

## 6. 요구사항

### 6.1 디자인 시스템 요구사항

1. 컬러 토큰은 `primary`, `secondary`, `success`, `warning`, `error`, `neutral`로 통합한다.
2. CSS 변수와 Tailwind 확장 컬러를 1:1 대응한다.
3. 타이포 스케일(`h1`, `h2`, `body`, `caption`)과 공통 line-height 규칙을 정의한다.
4. 공통 spacing 규칙(4px 기반)을 문서화하고 컴포넌트에 적용한다.

### 6.2 접근성 요구사항

1. 버튼/링크/입력 요소에 키보드 포커스 링을 명시한다.
2. 명도 대비는 WCAG AA 기준(일반 텍스트 4.5:1 이상)을 충족한다.
3. 인터랙티브 요소 중첩(예: `Link` 내부 `button`)을 제거한다.
4. 상태 메시지(성공/오류)는 시각적 구분 + 텍스트로 동시에 전달한다.

### 6.3 반응형 요구사항

1. 모달은 `max-w` + `w-full` 기반으로 모바일 가로 스크롤이 없어야 한다.
2. 포트폴리오 입력 테이블은 모바일에서 스택형 레이아웃으로 전환한다.
3. 주요 페이지(`home`, `stocks`, `backtest`, `result`)는 360px 기준에서 주요 작업이 가능해야 한다.

### 6.4 페이지별 요구사항

1. 홈/인증 페이지의 시각 톤을 헤더 및 내부 화면과 통일한다.
2. 백테스트 결과 차트는 기본 상태에서 불필요한 가로 스크롤이 없어야 한다.
3. 포트폴리오 상세/수정 화면은 동일한 정보 구조와 컴포넌트 룰을 공유한다.
4. 주식 상세 페이지의 가격 카드/기간 버튼 스타일을 공통 버튼 규칙과 맞춘다.

## 7. 기술 구현 원칙

1. 기존 Tailwind 기반을 유지한다.
2. 공통 UI 클래스는 `globals.css` 또는 컴포넌트 단으로 재사용 가능하게 정의한다.
3. 페이지별 임시 스타일을 줄이고 재사용 컴포넌트 중심으로 정리한다.
4. 기능 동작을 바꾸지 않는 디자인 변경을 우선한다.

## 8. 단계별 실행 계획

### Phase 1: 기반 정리 (2일)

- 토큰 단일화 (`globals.css`, `tailwind.config.ts`)
- 라이트/다크 테마 충돌 제거
- 공통 버튼/입력/카드 스타일 규칙 확정

### Phase 2: 레이아웃/내비게이션 정리 (2일)

- 헤더/푸터/메인 컨테이너 일관화
- 홈/로그인/회원가입 페이지 톤 정렬

### Phase 3: 핵심 플로우 개편 (4일)

- 백테스트 입력/모달 반응형 개선
- 포트폴리오 상세/수정 화면 구조 통합
- 결과 페이지 차트 가독성 개선

### Phase 4: 접근성/QA 마감 (2일)

- 키보드 탐색, 포커스, 명도 대비 점검
- 린트/기본 회귀 테스트
- 디자인 리뷰 피드백 반영

3. 반응형 개편된 주요 페이지
4. 접근성/QA 체크리스트 결과

## 10. 성공 지표(KPI)

1. 모바일(360px)에서 핵심 플로우 이탈 없이 완료 가능
2. UI 관련 이슈 재오픈율 감소(개편 전 대비 50% 이상)
3. 화면 간 스타일 불일치 피드백 감소
4. 기본 린트 오류 0, 주요 회귀 테스트 통과

## 11. 수용 기준(Definition of Done)

1. 고정폭 모달이 모바일에서 깨지지 않는다.
2. 백테스트/포트폴리오 입력 UI가 모바일에서도 조작 가능하다.
3. 전역 컬러/타이포 토큰이 단일 체계로 적용된다.
4. 시맨틱 위반(Link 내부 button 등)이 제거된다.
5. 결과 차트가 기본 뷰에서 가로 스크롤을 강제하지 않는다.
6. 린트와 주요 사용자 플로우 수동 테스트를 통과한다.
